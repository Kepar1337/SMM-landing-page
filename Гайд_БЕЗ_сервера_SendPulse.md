# Без сервера: як зробити, щоб пересланим посиланням не можна було обійти оплату

## Спершу — важлива правда про «унікальне посилання»

Посилання формату `https://t.me/cheat_code_bot?start=ZGw6MzI4NTIy` — це **службове посилання SendPulse «Link to launch a flow»**. Тут `ZGw6MzI4NTIy` — це base64 від `dl:328522`, тобто «запусти флоу №328522».

Тобто SendPulse справді генерує «хвіст», але це **ID флоу, а не одноразовий токен**. Таке посилання:

- **можна відкрити скільки завгодно разів;**
- **можна переслати** — і воно спрацює в іншого користувача.

> Одноразового/непересиланного посилання **без сервера не існує** — щоб «спалити» токен після першого використання, потрібне серверне сховище (це варіант із папки `/server`).

## Але твою мету можна вирішити без сервера

Мета: **щоб доступ був лише після оплати, а не за пересланим лінком.**

Рішення: зробити так, щоб посилання вело **не на контент, а на платіжний крок усередині бота**. Тоді хоч скільки ним ділись — кожен, хто відкриє, впреться в оплату, а курс отримає лише після **власної** успішної оплати. SendPulse перевіряє оплату сам (нативна інтеграція з WayForPay), тож обійти не вийде.

Простими словами: захищає не «унікальність лінка», а **платіжний шлагбаум у флоу**.

---

## Схема (без сервера)

```
Реклама → Лендінг (прогрів)
        │  кнопка «Відкрити доступ» → посилання SendPulse «Link to launch a flow»
        ▼
Telegram-бот: запускається ФЛОУ з офером
        │
   [ Кнопка оплати WayForPay ]  ← оплата ВСЕРЕДИНІ бота
        ├── Успішна оплата → видача міні-курсу (гілка «Успішна оплата»)
        └── Немає оплати   → нагадування / дожим
```

Пересланий лінк веде на цей самий флоу з оплатою — контент за ним не «витікає».

---

## Кроки налаштування в SendPulse

1. **Підключи WayForPay** до SendPulse: Account Settings → Accept payments → WayForPay → встав Merchant login + Secret Key (з кабінету WayForPay).
2. **Створи флоу продажу** (Flow builder):
   - **Message**: опис міні-курсу + ціна 790 грн.
   - **+ Add Button → Payment → Payment**:
     - спосіб оплати: **WayForPay**;
     - сума: **Fixed price = 790**, валюта UAH;
     - назва товару: «ЧИТ-КОД для SMM»;
     - *(опційно)* **Link expiration**: обмеж час дії платіжного посилання (напр. 6 год).
   - **Гілка «Успішна оплата» (зелена точка)**: постав тег `paid_chitkod` (Action) → повідомлення з видачею курсу (посилання/файли/відео, за днями).
   - **Гілка «Немає оплати» (синя точка)**: Pause → Filter «Payment complete» → дожим.
3. **Отримай посилання на цей флоу**: Bot structure → обери флоу → меню (⋯) → **«Link to launch a flow»**. Скопіюй посилання виду `https://t.me/ТВІЙ_БОТ?start=FLOW_ID`.
4. **Встав це посилання в лендінг**: у `index.html` → `CONFIG.botFlowUrl` (режим уже стоїть `payMode: 'bot'`).
5. **Протестуй**: відкрий лендінг → «Відкрити доступ» → бот → оплати тестово → переконайся, що курс приходить лише після оплати, а «гола» кнопка без оплати доступу не дає.

Готово — сервер не потрібен.

## Чесний трейд-оф (обери, що важливіше)

| Хочеш | Без сервера (режим `bot`) | З сервером (режим `server`, папка `/server`) |
|---|---|---|
| Оплата **на сайті** (форма WayForPay на лендінгу) | ❌ (оплата в боті) | ✅ |
| Працює **без сервера** | ✅ | ❌ (треба Node-хостинг) |
| Пересланим лінком **не обійти оплату** | ✅ (лінк = шлагбаум) | ✅ (одноразовий токен) |
| Справді **одноразове** посилання | ❌ (лінк багаторазовий, але веде на оплату) | ✅ |

**Рекомендація:** якщо не хочеш сервер — бери режим `bot`. Він повністю закриває головний ризик («поділився лінком → отримав курс безкоштовно»), бо ділитися можна лише посиланням на оплату.

Якщо ж принципова оплата саме на сайті **і** одноразове посилання — тоді потрібен сервер (усе вже готове в папці `/server`, дивись `server/README.md`).

## Піксель Facebook у режимі `bot`

Оскільки оплата в боті, подія `Purchase` на сайті не спрацює. Варіанти:
- на лендінгу лишаємо `PageView`, `ViewContent`, `InitiateCheckout` (клік «Відкрити доступ» = перехід у бота) — цього достатньо для оптимізації на початку;
- для точного `Purchase` — увімкни в SendPulse **Successful payment webhook** (Account Settings → API) і надсилай подію в Meta через Conversions API (окреме невелике налаштування).
