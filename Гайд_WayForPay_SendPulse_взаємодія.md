# WayForPay ↔ SendPulse: оплата з взаємодією платіжки та платформи

## Спершу — головне (чому саме так)

Ти хочеш, щоб **WayForPay передавав дані в SendPulse**, а SendPulse далі вів користувача в бот. Щоб це реально працювало, оплату має **ініціювати кнопка оплати SendPulse** (спосіб оплати — WayForPay). Причина:

- Коли людина платить на зовнішній сторінці WayForPay «сама по собі», SendPulse **не знає, який це Telegram-акаунт**, тож не може запустити для нього бот.
- Коли оплату запускає **кнопка SendPulse** — платформа знає підписника, отримує від WayForPay результат оплати нативно (взаємодія «платіжка ↔ платформа») і **сама веде користувача** гілкою «Успішна оплата».

Сторінка оплати при цьому — все одно WayForPay (картка, Apple/Google Pay, 3D-Secure). Відмінність лише в тому, що кнопка «Оплатити» стоїть у першому кроці бота, а не на самому сайті. Це єдиний спосіб отримати справжню взаємодію WayForPay↔SendPulse без власного сервера.

> Якщо ж потрібно, щоб форма WayForPay відкривалася прямо з сайту і при цьому оплата підтверджувалася — це вже вимагає власного сервера-посередника (варіант у папці `/server`). Без сервера взаємодія «платіжка ↔ SendPulse» можлива лише через кнопку оплати SendPulse.

## Схема потоку

```
Реклама FB/IG
      │
      ▼
Лендінг (кнопка «Відкрити доступ»)
      │  посилання «Link to launch a flow»
      ▼
Telegram-бот → запускається воронка (флоу) SendPulse
      │
   [ Повідомлення: офер + ціна 790 грн ]
      │
   [ Кнопка оплати WayForPay ]  ⇄  WayForPay (сторінка оплати)
      ├─ 🟢 Успішна оплата → тег paid_chitkod → ВИДАЧА міні-курсу
      └─ 🔵 Немає оплати   → Pause → Filter «Payment complete» → дожим
```

(Візуальну схему цього потоку я показав окремо в чаті.)

---

## Покрокове налаштування

### Крок 1. Підключити WayForPay до SendPulse
1. У WayForPay: **Налаштування магазину → Реквізити мерчанта** → скопіюй **Merchant login** і **Merchant Secret Key**.
2. У SendPulse: **Account Settings → Accept payments → WayForPay → Connect** → встав ці два значення, обери валюту **UAH** → **Save**. Статус має стати **Connected**.

### Крок 2. Підготувати змінні/теги (за бажанням)
- У SendPulse → **Audience** створи тег `paid_chitkod` (позначати покупців) — знадобиться для сегментів і розсилок.

### Крок 3. Зібрати воронку (Flow builder)
1. **Тригер флоу.** Це твій вітальний ланцюжок (той, що з посиланням `?start=6a4432fdbd262802b6079368`).
2. **Message — офер:** короткий опис міні-курсу + ціна. Можна додати картинку.
3. **Кнопка оплати:** у цьому ж елементі **+ Add Button → Payment → тип Payment**:
   - **Спосіб оплати:** WayForPay (обери мерчанта);
   - **Сума:** Fixed price = **790**, валюта UAH;
   - **Назва товару:** «ЧИТ-КОД для SMM»;
   - *(опційно)* **Link expiration:** термін дії платіжного посилання (напр. 6 годин).
4. **Гілка «Успішна оплата» (зелена точка кнопки):**
   - **Action:** признач тег `paid_chitkod`;
   - далі — повідомлення з видачею міні-курсу (уроки/файли/посилання, за днями за потреби).
5. **Гілка «Немає оплати» (синя точка):**
   - **Pause** (скільки чекати оплату) → **Filter: Payment complete** для цього товару → гілка «не оплатив» = нагадування/дожим (можна зациклити з повторною паузою).
6. **Save** і запусти флоу.

### Крок 4. Звʼязати з лендінгом
1. У SendPulse: **Bot structure** → цей флоу → меню (⋯) → **«Link to launch a flow»** → скопіюй посилання `https://t.me/smmcheatcode_bot?start=6a4432fdbd262802b6079368`.
2. У `index.html` це вже вставлено: `CONFIG.payMode: 'bot'`, `CONFIG.botFlowUrl` = це посилання. Кнопки лендінгу ведуть на нього.

### Крок 5. Протестувати
1. Відкрий лендінг → «Відкрити доступ» → відкривається бот → тисни кнопку оплати.
2. На тестовій сторінці WayForPay обери **Test checkout → Completed**.
3. Переконайся, що спрацювала гілка «Успішна оплата» і прийшов міні-курс.
4. Історія оплат: **Chats → Orders** та **Account Settings → Accept payments**.

## Як це виглядає в SendPulse (елементи флоу)

| Крок у флоу | Елемент SendPulse | Налаштування |
|---|---|---|
| Офер | **Message** | текст + картинка |
| Оплата | **Button → Payment** | WayForPay, Fixed price 790 UAH, товар «ЧИТ-КОД для SMM» |
| 🟢 після оплати | зелена точка кнопки | **Action** (тег `paid_chitkod`) → повідомлення з курсом |
| 🔵 без оплати | синя точка кнопки | **Pause** → **Filter: Payment complete** → дожим |

## Часті питання

**А якщо людина вже оплатила і знову відкриє бот?**
Гілка «Успішна оплата» чекає підтвердження до 24 год; за фільтром «Payment complete» можна не показувати повторний запит на оплату вже наявним покупцям.

**Чи можна лишити оплату прямо на сайті?**
Так, але тоді WayForPay не «спілкується» із SendPulse (нема кому передати, який це Telegram-користувач). Варіанти:
- без сервера — браузер сам редіректить у бот після оплати (режим `wfp` у `CONFIG`, але без підтвердження оплати);
- з підтвердженням — власний сервер-посередник (папка `/server`).

**Пікселя Facebook «Purchase».**
У цій схемі оплата в боті, тож `Purchase` на сайті не спрацює. Для точності ввімкни в SendPulse **Successful payment webhook** (Account Settings → API) і надсилай подію в Meta через Conversions API.
